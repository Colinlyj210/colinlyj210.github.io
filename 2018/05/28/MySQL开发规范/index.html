MySQL开发规范

发表于 2018-05-28

MySQL开发规范

1.命名规范

(1)库名、表名、字段名必须使用小写字母，并采用下划线分割。

(2)库名、表名、字段名尽量不用MySQL保留字。

（常见的 date, day, month, year, year_month, hour_minute, hour_second, user, host, hosts, owner, names, type, view, begin, end, status, event, page ）。

2.基础规范

(1)使用InnoDB存储引擎。

(2)表字符集使用UTF8，必要时可使用UTF8MB4字符集。

(3)所有表都需要添加注释；不能见名知意的字段都需要增加注释。

3.库表设计

(1)将大字段、访问频率低的字段拆分到单独的表中存储，分离冷热数据。

4.字段设计

(1)建议使用UNSIGNED存储非负数值。

(2)建议使用INT UNSIGNED存储IPV4。

(3)用DECIMAL代替FLOAT和DOUBLE存储精确浮点数，例如支付相关数据。

(4)INT类型固定占4字节存储，例如INT(4)仅代表显示字符宽度为4位，不代表存储长度，一般不用写显示宽度，不写保留默认的即可。

(5)区分使用TINYINT、SMALLINT、MEDIUMINT、INT、BIGINT数据类型。例如取值范围为0-80时，使用TINYINT。

类型 大小 范围（有符号） 范围（无符号） 用途
TINYINT 1 字节 (-128，127) (0，255) 小整数值
SMALLINT 2 字节 (-32 768，32 767) (0，65 535) 大整数值
MEDIUMINT 3 字节 (-8 388 608，8 388 607) (0，16 777 215) 大整数值
INT或INTEGER 4 字节 (-2 147 483 648，2 147 483 647) (0，4 294 967 295) 大整数值
BIGINT 8 字节 (-9 233 372 036 854 775 808，9 223 372 036 854 775 807) (0，18 446 744 073 709 551 615) 极大整数值

(6)建议使用TINYINT来代替ENUM, BIT类型。

(7)尽可能不使用TEXT、BLOB类型。

(8)使用VARBINARY存储大小写敏感的变长字符串或二进制内容。

(9)使用尽可能小的VARCHAR字段。VARCHAR(N)中的N表示字符数而非字节数。

5.索引规范

(1) 索引名必须全部使用小写。

(2) 非唯一索引按照“idx表名首字母简写字段名首字母简写_字段名首字母简写…”进行命名，

唯一索引按照“uniq表名首字母简写字段名首字母简写_字段名首字母简写…”进行命名，

简写尽量不多于4个字符，表名简写尽量库内不重名即可，字段名简写尽量表内不重复即可。

(3)表必须有主键，主键字段放首行，推荐使用自增列作为主键，当表可能较大时主键优先考虑使用bigint。

(4)禁止冗余和重复索引。

(5)尽量不使用外键，可以考虑代码实现。

(6)关联表查询时，JOIN列的数据类型必须相同，并且要建立索引。

(7)组合索引中，应优先考虑分组和排序需求，在同等条件下，区分度大的字段放在最前。

(8)对字符串使用前缀索引，前缀索引长度不超过8个字符，但无法分组排序。

(9)不对过长的VARCHAR字段建立索引。建议优先考虑添加CRC32或MD5伪列，并对伪列建⽴索引。

(10)合理创建组合索引，(a，b，c) 相当于 (a) 、(a，b) 、(a，b，c)。

(11)合理使用覆盖索引减少IO，避免排序。

6.SQL设计

(1)用IN代替OR。SQL语句中IN包含的值不应过多，尽量少于1000个。

(2)禁止隐式转换。数值类型禁止加引号；字符串类型必须加引号。

(3)子查询性能差时可以尝试用JOIN代替。

(4)避免在MySQL条件字段中进行数学运算和函数运算。

(5)减少与数据库交互次数，尽量采用批量SQL语句。

a) INSERT ... ON DUPLICATE KEY UPDATE 
b) REPLACE INTO 
c) INSERT IGNORE 
d) INSERT INTO VALUES(),()...

(6)对同一个表的多次alter操作必须合并为一次操作。

(7)拆分复杂SQL为多个小SQL，避免大事务。

(8)用UNION ALL代替UNION，UNION去重，UNION ALL不去重。

(9)统计行数用COUNT(*)或COUNT(1)。

(10)INSERT语句必须指定字段列表，禁止使用 INSERT INTO TABLE VALUES(1, …)，必须使用INSERT INTO TABLE(a, …) VALUES(1, …)。

(11)禁止单条SQL语句同时更新多个表。

(12)使用%前导查询无法使用索引，例如：like “%abc”，大结果集不用，小结果集可以使用。

(13)使用EXPLAIN判断SQL语句是否合理使用索引，尽量避免extra列出现：Using File Sort、Using Temporary。

(14)如果不需要排序，应该去掉order by。

7.行为规范

(1)表结构变更需要通知DBA进行审核。

(2)提交线上建表改表需求，尽可能详细注明涉及到的所有SQL语句（包括INSERT、DELETE、UPDATE），便于DBA进⾏审核和优化。

(3)查询结果集较大时需要加上limit, 避免把sgw带宽占满影响其他人访问。

(4)客户端运行的sql被取消或者停止后，还需要再执行show processlist，如果还在运行，则kill掉该sql。
